# 编辑模式与权限（Draft）

> 目标：在“透明可溯源 + 可回滚”的前提下，提供可选的编辑模式（如上帝/沉浸式），用来决定用户能编辑哪些数据/参数/条目，从而兼顾“可控”与“沉浸感”。
>
> 本文为需求/设计说明，不做工程落地映射。
>
> 最后更新：2026-01-07

---

## 0. 为什么需要“模式”

同一套系统在不同心态下会有冲突需求：

- **创作/编辑心态**：希望像写作软件一样随时修正设定、改条目、调权重。
- **沉浸/游玩心态**：不希望频繁进入“后台管理”，否则会破坏跑团/互动小说的沉浸感。

所以建议把“可编辑范围”抽象成模式；模式本质是 **权限与 UI 暴露策略**，而不是不同的数据模型。

---

## 1. 数据类型分层（决定哪些应该可改）

为了避免“什么都能改导致不可复现”，建议把数据分成 4 类：

1) **Canon Facts（已确认事实）**
   - 例：角色外观关键字段（已确认）、已发生的关键事件（已确认）、世界规则（已确认）。
   - 特点：会直接约束模型行为；改动风险最高。

2) **Draft Judgements（模型判断/推测）**
   - 例：伏笔 Tracker（证据积累/回收接近/置信度）、关联器 Link 的正负与 delta、重要性评分、风险度。
   - 特点：本质是“系统辅助判断”，用户应能纠正；默认可回滚。

3) **Config / Budget（系统配置与预算）**
   - 例：模块启用状态、注入预算、View 的 top‑N、Tracker 权重。
   - 特点：影响全局 token/行为；适合集中放在“高级设置”。

4) **Derived/Computed（派生字段）**
   - 例：`urgency_score`、综合进度 `progress`（由两轴聚合）。
   - 特点：建议只读；若要允许手动改，应该用“manual_override 字段”而不是直接改计算结果。

---

## 2. 模式建议（v0）

### 2.1 Mode A：上帝/编辑模式（God / Editor）

**目标**：最大可控、最大自由。

- 可编辑：Draft + Config（几乎全部）
- Canon：允许直接编辑，但必须：
  - 二次确认（“这会改变事实约束”）
  - 自动生成版本记录与回滚点
  - 强制填写/选择“修改原因”（纠错/剧情变化/手工修订）

适用：创作、重构设定、复盘整理、补档。

### 2.2 Mode B：标准模式（Standard / Human‑in‑the‑loop）

**目标**：你当前最符合的默认模式。

- Draft：可编辑（包括 Link delta、Tracker 权重、误解/主观记忆等）
- Canon：不允许直接编辑；必须走 Proposal 流程（允许/拒绝/编辑后允许）
- Config：可编辑，但建议收在“高级设置”，默认不打断对话

适用：大多数日常跑团/互动小说。

### 2.3 Mode C：沉浸模式（Immersive）

**目标**：尽量不打断故事体验。

- Draft：不允许直接编辑；只允许：
  - 接受/拒绝系统 Proposal
  - “询问 agent 给建议”（但仍需用户最后确认）
  - 一键回滚最近一次更新
- 对话历史：✅ 允许编辑历史消息，且默认显示入口（用于快速纠错/修复模型抽风）；编辑会触发派生数据失效与重算，并记录日志
- Canon：不允许直接编辑（同 Standard）
- Config：大部分隐藏；只保留最少开关（例如启用/禁用某模块）

适用：用户只想“玩故事”，不想管理后台。

## 3. 权限矩阵（按模块/块给一个直觉表）

> 这里不是最终 UI 设计，只是把边界写清楚，避免后续反复推翻。

- `timeline`（关键事件）
  - Standard：系统判定为主；用户可“标记关键/非关键”（你已确认）
  - Immersive：只允许接受/拒绝“关键事件提议”（不直接编辑内容）
- `foreshadow_hooks`（伏笔）
  - Standard：允许编辑 Link 的 `delta`、relation（正/负/无关）、Tracker 权重；但都算 Draft
  - Immersive：不允许改 delta；只允许“接受/拒绝关联提议”
- `style`
  - Standard：用户可编辑 Style 卡片
  - Immersive：可编辑（因为它是“写作体验设置”，但可限制频率/范围）
- `state`
  - Standard：允许修正状态条目（例如物品、伤势），但写入 Canon 仍需确认
  - Immersive：只允许接受/拒绝状态变更提议

> 注意：你已确认“模块禁用后禁止编辑”。模式只在“模块启用”时生效。

---

## 4. 建议的默认值

- 默认模式：**Standard**
- 模式切换：允许随时切换（不会改数据，只改“可编辑范围/可见性/触发弹窗策略”）
- 高风险编辑（Canon）：永远需要版本记录与回滚点

---

## 5. 仍需你确认的 2 个选择

1) ✅ 已确认：默认向用户暴露 **Standard + Immersive**；`God/Editor` 放在高级开关里。
2) ✅ 已确认：沉浸模式下 **允许编辑 Style**（它不改变事实，只改变叙事体验）。
