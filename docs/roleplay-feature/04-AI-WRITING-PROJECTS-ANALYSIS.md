## 2. MuMuAINovel 核心架构

### 2.1 RTCO 框架（上下文优先级）

> 这里的 RTCO 更接近 “Role / Task / Context / Output” 的结构化提示词组装范式，并辅以 `P0/P1/P2` 的优先级标注。

```
P0 - 核心（必须包含）
├── 章节大纲
├── 衔接锚点（上一章结尾）
└── 字数要求

P1 - 重要（按需包含）
├── 涉及角色信息
├── 情感基调
└── 写作风格

P2 - 参考（条件触发）
├── 相关记忆（语义检索）
├── 故事骨架（50章+）
└── 伏笔提醒
```

### 2.2 动态上下文裁剪

根据章节序号自动调整上下文复杂度：

| 章节范围 | 衔接长度 | 记忆数量 | 额外内容 |
|----------|----------|----------|----------|
| 第 1 章 | 无 | 无 | 仅大纲+角色 |
| 第 2-10 章 | 300 字 | 无 | 涉及角色 |
| 第 11-50 章 | 500 字 | 3 条 | 相关记忆 |
| 第 51+ 章 | 500 字 | 5 条 | 故事骨架 |

**核心代码逻辑**：
```python
ENDING_LENGTH_SHORT = 300    # 1-10章：短衔接
ENDING_LENGTH_NORMAL = 500   # 11章+：标准衔接
MEMORY_COUNT_LIGHT = 3       # 11-50章：轻量记忆
MEMORY_COUNT_FULL = 5        # 51章+：完整记忆
SKELETON_THRESHOLD = 50      # 启用故事骨架的章节阈值
```

### 2.3 记忆系统（StoryMemory）

**记忆类型分类**：
| 类型 | 说明 | 用途 |
|------|------|------|
| `plot_point` | 情节点 | 关键剧情转折 |
| `character_event` | 角色事件 | 角色行为/变化 |
| `world_detail` | 世界观细节 | 设定补充 |
| `hook` | 钩子 | 悬念/冲突点 |
| `foreshadow` | 伏笔 | 待回收的伏笔 |
| `dialogue` | 重要对话 | 关键对话 |
| `scene` | 场景描写 | 氛围/环境 |

**伏笔管理**：
```python
is_foreshadow: int
# 0 = 普通记忆
# 1 = 已埋下伏笔（待回收）
# 2 = 伏笔已回收

foreshadow_resolved_at: str  # 回收的章节 ID
foreshadow_strength: float   # 伏笔强度 0.0-1.0
```

**智能检索策略**：
1. 语义检索相关记忆（提高重要性阈值 0.7+）
2. 检查即将到期的伏笔（lookahead=5 章）
3. 合并并格式化，限制最大长度 500 字

### 2.4 剧情分析（PlotAnalysis）

**结构化分析维度**：
- **剧情阶段**：开端 / 发展 / 高潮 / 结局 / 过渡
- **冲突分析**：类型（人与人/人与己/人与环境）+ 强度 1-10
- **情感曲线**：`{start: 0.3, middle: 0.7, end: 0.5}`
- **钩子列表**：悬念/情感/冲突/认知 + 强度 + 位置
- **角色状态追踪**：state_before → state_after + 触发事件

**质量评分体系**：
- overall_quality_score（整体质量）
- pacing_score（节奏）
- engagement_score（吸引力）
- coherence_score（连贯性）

## 3. Arboris-Novel 核心架构

### 3.1 五层信息架构

```
┌─────────────────────────────────────────────────────────────┐
│ L1 世界蓝图   │ JSON   │ 世界设定、人物档案、规则      │ ❌ 不检索 │
├─────────────────────────────────────────────────────────────┤
│ L2 剧情记忆   │ 向量库  │ 已生成章节的分块文本          │ ✅ RAG   │
├─────────────────────────────────────────────────────────────┤
│ L3 章节摘要   │ JSON   │ 章节标题 + 摘要 + 主要人物    │ ✅ RAG   │
├─────────────────────────────────────────────────────────────┤
│ L4 上下文桥接 │ Markdown │ 上一章摘要 + 结尾 500 字    │ ✅ 固定  │
├─────────────────────────────────────────────────────────────┤
│ L5 当前输入   │ 自然语言 │ 标题、摘要、写作指令         │ ❌ 不检索 │
└─────────────────────────────────────────────────────────────┘
```

**设计原则**：
- L1（蓝图）作为**固定约束**，保证逻辑一致性
- L2-L3 通过 **RAG 检索**动态召回相关内容
- L4 保证**衔接自然**，情绪与逻辑连贯

### 3.2 向量化与 RAG 系统

**切分策略**：
```python
chunk_size = 300-600  # 字符
chunk_overlap = 80-130  # 字符
# 优先使用 LangChain RecursiveCharacterTextSplitter
# 按段落/句子逐级回退切分
```

**Metadata 设计**：
```python
{
    "chapter_number": 12,
    "chapter_title": "暗夜追踪",
    "chunk_id": "ch12_chunk_03",
    "tags": ["张三", "酒馆", "冲突"]
}
```

**检索策略**：
- 剧情记忆层：top-K = 5
- 章节摘要层：top-K = 3-5
- 动态调整：章节少 → K 大，章节多 → K 小

### 3.3 工作流程

```
项目创建 → 概念对话 → 蓝图生成 → 章节生成 → 版本评审
     ↑                                      ↓
 （持续迭代） ← 手动编辑 ← 向量入库 ← 摘要提取 ← RAG 支撑下一章
```

**各阶段模型参数**：
| 阶段 | 温度 | 说明 |
|------|------|------|
| 概念对话 | 0.8 | 引导发散思维 |
| 蓝图生成 | 0.3 | 保持结构稳定 |
| 章节生成 | 0.9 | 鼓励创意表达 |
| 章节评审 | 0.3 | 理性分析 |
| 摘要提取 | 0.15 | 精确提炼 |

### 3.4 "人味"写作提示词设计

**核心原则**：
> "写作时，把自己当成一个讲故事的人，而不是一个执行任务的程序。"

**反 AI 味技法**：

1. **语言呼吸感**
   - 短句长句自然交替
   - 偶尔用不完整句子
   - 避免"显而易见"、"毋庸置疑"等 AI 常用词

2. **角色不完美**
   - 说话说一半忘记
   - 紧张时做无意义小动作
   - 情绪冲昏头脑，做出不理智决定

3. **细节偏执**
   - 不用"温暖的阳光"
   - 用"阳光照在脸上，让人想起小时候发烧时盖的那床毛毯"

4. **情绪复杂性**
   - 愤怒里有委屈
   - 悲伤里有解脱
   - 快乐里有一丝不安

5. **对话潜台词**
   - "你还好吗？" → "你还爱我吗？"
   - "随便你。" → "你敢试试看。"

**反 AI 味检查清单**：
- [ ] 这段话听起来像教科书吗？
- [ ] 用了太多"然而""因此""显然"吗？
- [ ] 角色反应太完美、太理智了吗？
- [ ] 有没有让人意外但又合理的细节？
- [ ] 读起来有节奏感吗？

## 4. 共性提炼：优秀创作系统的核心要素

### 4.1 分层上下文管理

| 层级 | 内容 | 处理方式 | 目的 |
|------|------|----------|------|
| 固定层 | 世界观、角色设定 | 始终注入 | 保证一致性 |
| 动态层 | 相关记忆、历史事件 | RAG 检索 | 按需召回 |
| 桥接层 | 上一章摘要+结尾 | 固定格式 | 保持衔接 |
| 任务层 | 当前章节目标 | 用户输入 | 明确方向 |

### 4.2 智能记忆系统

**核心能力**：
1. **结构化存储**：按类型分类（情节/角色/伏笔等）
2. **重要性评分**：过滤低质量记忆
3. **语义检索**：基于当前上下文召回相关内容
4. **伏笔追踪**：埋下 → 提醒 → 回收

### 4.3 动态裁剪策略

**核心思想**：根据故事进度自动调整上下文复杂度
- 早期章节：简单，聚焦角色和大纲
- 中期章节：加入记忆检索
- 后期章节：故事骨架 + 智能记忆

### 4.4 多版本评审机制

**流程**：
1. 生成多个候选版本（N 个，例如 3；可配置）
2. AI 评审给出改进建议
3. 用户选择或编辑
4. 确认后生成摘要并入库

## 5. 对 ChatBoxApp 的启示

### 5.1 可直接借鉴

| 特性 | 来源 | 适用场景 |
|------|------|----------|
| 分层上下文（L1-L5） | Arboris | 角色扮演 |
| 动态裁剪策略 | MuMu | 长对话管理 |
| 伏笔追踪 | MuMu | 创作场景 |
| "人味"提示词 | Arboris | 所有场景 |
| 衔接锚点（500字） | 两者 | 上下文桥接 |

### 5.2 需要简化

| 原特性 | 简化方案 |
|--------|----------|
| 完整 RAG 系统 | 先用关键词匹配（类似 World Info） |
| 多版本生成 | 单版本 + 重新生成 |
| 剧情分析 | 可选功能 |
| 向量数据库 | 先用本地存储 |

### 5.3 创新方向

1. **角色记忆系统**：为每个角色维护独立的"记忆"
2. **情感状态追踪**：角色当前情绪影响回复风格
3. **场景感知**：自动识别并注入场景氛围
4. **写作风格模板**：预设多种创作风格（文艺/网文/严肃等）
