## [3.4.0] - 2025-10-28 📎 附件功能全面优化

### ✨ 新增特性

#### 1. 📸 图片查看器（支持手势缩放）

**新功能**：
- ✅ 点击气泡中的图片全屏查看
- ✅ 支持手势缩放（pinch-to-zoom）
- ✅ 支持双指平移浏览

#### 2. 🎯 附件快照系统（轻量化存储）

**架构优化**：
- ✅ 创建 `AttachedFileSnapshot` 轻量级快照类
- ✅ 只保存文件路径引用，不复制文件内容
- ✅ 支持图片和文档/代码文件统一处理
- ✅ 消息持久化时自动创建快照

**数据模型**：
```dart
class AttachedFileSnapshot {
  final String id;
  final String name;
  final String path;
  final String mimeType;
  final FileType type;
}
```

**Message 模型更新**：
- `attachedFileIds` (List<String>) → `attachedFiles` (List<AttachedFileSnapshot>)
- 完整保存附件元数据，避免丢失引用

#### 3. 🔍 文件存在性检测（优雅降级）

**智能处理**：
- ✅ 使用 FutureBuilder 异步检查文件是否存在
- ✅ **图片文件**：
  - 存在：显示缩略图 + 点击查看原图
  - 不存在：显示占位符（"图片已不存在"）
- ✅ **文档/代码文件**：
  - 存在：显示可点击的文件卡片
  - 不存在：显示占位符（"文件已不存在"）

**占位符设计**：
- 灰色背景 + 删除线文件名
- 明确的错误提示信息
- 统一的 UI 风格

#### 4. 🎭 附件栏优雅消失逻辑

**用户体验优化**：
```

  ↓
附件栏立即消失（瞬间反馈）
  ↓
API 调用中（附件数据仍在内存）
  ↓
成功：清空数据 + 恢复可见性状态
失败：附件栏重新显示（可重试）
```

**技术实现**：
- 新增 `_attachmentBarVisible` 独立状态
- 发送时隐藏 UI，但保留数据
- 成功后才真正清空 `attachedFiles`
- 失败时自动恢复显示，用户可重试

**流程对比**：
| 阶段 | 旧逻辑 | 新逻辑 |
|------|--------|--------|
| 点击发送 | 附件栏保留 | ✅ 附件栏瞬间消失 |
| API 调用中 | 附件栏可见 | ✅ 附件栏隐藏（清爽）|
| 发送成功 | 附件栏消失 | ✅ 清空数据 |
| 发送失败 | 手动重试 | ✅ 附件栏恢复（可重试）|

```

    A[用户访问] --> B[输入账号密码]
```

## [3.2.1] - 2025-10-27 🐛 自定义助手分组 Bug 修复

### 🐛 Bug 修复

#### 关键修复：自定义助手对话分组错误

**问题描述**：
- 创建自定义助手后，在该助手下新建对话
- 对话被错误地归入其他助手（如"默认助手"）
- 用户无法正确管理和组织对话

**根本原因**：
- 旧逻辑使用 `systemPrompt` 字符串匹配角色
- 空 systemPrompt 或相同 systemPrompt 导致匹配失败
- 编辑角色后，旧对话无法正确匹配

**修复方案**：
- ✅ 新增 `roleId` 和 `roleType` 字段到 Conversation 模型
- ✅ 使用角色 ID 进行精确匹配，而不是 systemPrompt
- ✅ 区分内置角色（'preset'）和自定义角色（'custom'）
- ✅ 完全向后兼容旧数据（自动回退到 systemPrompt 匹配）

### 🔧 技术改进

**数据模型增强**：
```dart
class Conversation {
  // 新增字段
  String? roleId;     // 角色ID（精确匹配）
  String? roleType;   // 'preset' 或 'custom'
}
```

**分组逻辑优化**：
- 优先使用 roleId + roleType 精确匹配
- 支持角色被删除后的优雅降级
- 旧数据自动使用 systemPrompt 匹配

**修改的文件**：
- `lib/models/conversation.dart` - 添加新字段
- `lib/services/conversation_service.dart` - 更新创建逻辑
- `lib/pages/chat_page.dart` - 保存角色ID
- `lib/widgets/conversation_drawer.dart` - 优化分组匹配

### ✅ 修复效果

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| 新建自定义助手对话 | ❌ 归入错误助手 | ✅ 正确归组 |
| 空 systemPrompt | ❌ 匹配失败 | ✅ 正常工作 |
| 编辑角色后 | ❌ 旧对话丢失 | ✅ 仍能匹配 |

## [3.2.0] - 2025-10-27 💾 滚动位置持久化

### ✨ 新增特性

#### 📍 滚动位置持久化
- **应用重启后恢复滚动位置** - 每个会话的滚动位置现在会保存到本地存储
- **完美的用户体验** - 关闭应用再打开时，自动恢复到之前的阅读位置
- **智能保存** - 滚动时自动保存位置，防止频繁写入（500ms 防抖）
- **可靠恢复** - 应用启动时自动恢复到上次的滚动位置

### 🔧 技术实现

**数据模型更新**：
```dart
class Conversation {
  // ... 现有字段
  int? scrollIndex; // 新增：保存滚动位置的消息索引
}
```

**滚动位置监听**：
- 使用 `ItemPositionsListener` 监听滚动位置变化
- 自动保存第一个可见消息的索引
- 延迟保存机制（500ms）避免频繁写入

**位置恢复**：
- 应用启动时在 `initState` 中自动恢复
- 使用 `ItemScrollController.jumpTo()` 精确跳转
- 验证索引有效性，防止越界

### 🎯 解决的问题

**之前的行为**：
- ❌ 应用重启后总是回到对话顶部
- ❌ 丢失阅读进度
- ❌ 需要手动滚动回之前的位置

**现在的行为**：
- ✅ 应用重启后精确恢复滚动位置
- ✅ 保持阅读进度
- ✅ 无缝继续阅读

### 📝 使用说明

**自动工作**：
- 功能完全自动化，无需用户操作
- 滚动时自动保存位置
- 应用启动时自动恢复位置

**适用场景**：
- ✅ 会话切换（内存中保持）
- ✅ 应用热重载（开发时）
- ✅ 应用完全重启
- ✅ 应用关闭后重新打开

### 🔍 技术细节

**保存策略**：
1. 监听 `ItemPositionsListener` 的位置变化
2. 获取第一个可见消息的索引
3. 只在位置真正改变时保存
4. 延迟 500ms 后触发持久化（防抖）

**恢复策略**：
1. 从 `Conversation.scrollIndex` 读取保存的位置
2. 在 `postFrameCallback` 中恢复（确保 UI 已构建）
3. 验证索引有效性（0 ≤ index ≤ 消息总数）
4. 使用 `jumpTo` 立即跳转到目标位置

**性能优化**：
- 防抖机制减少写入频率
- 只在位置真正改变时保存
- 使用 `jumpTo` 而非 `scrollTo`（无动画，更快）

### 📊 影响范围

**修改的文件**：
- `lib/models/conversation.dart` - 添加 `scrollIndex` 字段
- `lib/widgets/conversation_view.dart` - 添加监听器和恢复逻辑

**向后兼容性**：
- ✅ 完全向后兼容
- ✅ 旧数据自动兼容（scrollIndex 为 null 时不影响）
- ✅ 不影响现有功能
