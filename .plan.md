|----|------|--------|

| **Sleeptime 集成** | 空闲任务调度 | P2 |

### 1.3 架构差异分析

**M4 Spec 设计**：
```
impl/
├── scene_detector.dart     # 独立 Handler 类
├── state_updater.dart
├── key_event_extractor.dart
└── consistency_heavy.dart
```

**当前实现**：
- Handler 逻辑 **内置于 ProposalTransformer**
- `SceneDetectorTransformer.transform()` 完成了 `SceneDetector.handle()` 的职责
- 无需额外的 `impl/` 目录

**结论**：当前实现是 M4 Spec 的**简化版本**，功能等效但更精简。

## 三、决策点

### DP-1: 是否需要 `impl/` 目录？

**选项**：
- A) **保持现状**：Handler 逻辑内置于 Transformer（推荐）
- B) 按 M4 Spec 重构：创建独立 Handler 类

**分析**：
- 当前实现功能完整，无需额外抽象
- 独立 Handler 类增加复杂度，无明显收益
- **建议**：选项 A，更新 M4 Spec 文档以反映实际实现

### DP-2: 测试策略

**选项**：
- A) **Mock LLM 调用**（推荐）：使用预定义响应
- B) 真实 LLM 调用：依赖实际 API

**分析**：
- Mock 测试快速、可重复、无成本
- 真实调用适合少量 E2E 验证
- **建议**：Phase 1-2 用 Mock，Phase 3 可选真实验证
